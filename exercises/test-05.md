## 一、选择题（每题5分，共20分）

### 1. 预训练模型需要指令微调的原因是？


A. 
预训练模型的参数太少

B. 
预训练模型不知道什么是"指令"

C. 
预训练模型速度太慢

D. 
预训练模型太大了

<details>
<summary>💡 点击查看答案</summary>

**答案：B**

预训练模型学会了语言规律，但不知道什么是"指令"，不知道要完成任务。
</details>

### 2. 指令数据的三个要素是？


A. 
输入、输出、标签

B. 
instruction、input、output

C. 
问题、答案、分数

D. 
数据、模型、训练

<details>
<summary>💡 点击查看答案</summary>

指令数据包含：instruction（指令）、input（输入）、output（输出）三部分。
</details>

### 3. LoRA 的主要优势是？


A. 
训练更快

B. 
只需训练少量参数

C. 
效果更好

D. 
不需要数据

<details>
<summary>💡 点击查看答案</summary>

**答案：B**

LoRA（低秩适配）只需训练少量参数，大幅降低训练成本。
</details>

---

### 4. 高质量指令数据的标准不包括？


A. 
指令清晰明确

B. 
任务多样化

C. 
答案准确有用

D. 
数据量必须很大

<details>
<summary>💡 点击查看答案</summary>

**答案：D**

研究表明，几万条高质量指令就足够，数据量不是越多越好，质量更重要。
</details>

## 三、简答题（每题15分，共30分）

### 7. 设计一条翻译任务的指令数据（包含 instruction、input、output）。

<details>
<summary>💡 点击查看参考答案</summary>

**参考答案：**

```json
{
  "instruction": "请将以下中文句子翻译成英语",
  "input": "人工智能正在改变世界",
  "output": "Artificial intelligence is changing the world."
}
```

- input 是待翻译内容

</details>

### 8. 解释 LoRA 为什么能降低微调成本。

<details>
<summary>💡 点击查看参考答案</summary>

**参考答案：**

LoRA 的核心思想：

1. **不修改原参数**：保持预训练模型参数不变
2. **添加小模块**：只添加少量可训练参数
3. **低秩分解**：用低秩矩阵近似参数更新

**类比：**
- 全量微调 = 重新装修整个房子
- LoRA = 只换几个关键位置的家具

**效果：**
- 参数量减少：可能只有原来的 1% 或更少
- 内存需求降低：不需要存储所有梯度
- 训练更快：计算量大幅减少
- 效果接近：多数情况下效果相当
</details>

## 四、设计题（30分）

### 9. 为"数学题解答"任务设计 3 条指令数据。

要求：
1. 包含不同难度（简单、中等、困难）
2. 每条数据包含 instruction、input、output
3. output 要展示解答过程

<details>
<summary>💡 点击查看参考答案</summary>

**参考答案：**

**简单：**
```json
{
  "instruction": "请解答以下数学题，写出计算过程",
  "input": "3 + 5 = ?",
  "output": "解答：3 + 5 = 8\n答案：8"
}
```

**中等：**
```json
{
  "instruction": "请解答以下数学题，写出计算过程",
  "input": "一个长方形的长是8厘米，宽是5厘米，求面积",
  "output": "解答过程：\n1. 长方形面积公式：面积 = 长 × 宽\n2. 代入数值：面积 = 8 × 5 = 40\n答案：40平方厘米"
}
```

**困难：**
```json
{
  "instruction": "请解答以下数学题，写出计算过程",
  "input": "小明有苹果的数量是小红的2倍，如果小红有x个苹果，两人一共有多少个？当x=5时，答案是多少？",
  "output": "解答过程：\n1. 设小红有 x 个苹果\n2. 小明有 2x 个苹果\n3. 两人一共有：x + 2x = 3x 个\n4. 当 x = 5 时：3 × 5 = 15\n答案：一共有3x个苹果，当x=5时是15个"
}
```
</details>
