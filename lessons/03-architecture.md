## 📖 第一部分：注意力机制

### 1.1 人类的注意力

**上课时，你会注意什么？**

```
👨‍🏫 老师的讲解 ────→ 重要！认真听
📝 黑板上的板书 ────→ 重要！记下来
🔔 同学的提问 ────→ 一般关注
🌧️ 窗外的雨声 ────→ 忽略
```

**关键洞察：** 不是所有信息都同等重要，我们会"分配注意力"！

### 1.2 AI 的注意力

**问题：** 如何让 AI 知道哪些信息重要？

**例子：翻译 "我 喜欢 苹果"**

```
翻译到 "I" 时：
  关注 "我" ────→ 100%
  
翻译到 "like" 时：
  关注 "喜欢" ────→ 90%
  关注 "我" ────→ 10%（知道是谁喜欢）
  
翻译到 "apples" 时：
  关注 "苹果" ────→ 95%
  关注 "喜欢" ────→ 5%（知道是喜欢什么）
```

### 1.3 注意力三要素：Q、K、V

**用图书馆找书类比：**

```
┌─────────────────────────────────────────────────┐
│              图书馆找书 = 注意力机制              │
├─────────────────────────────────────────────────┤
│                                                 │
│  🔍 Query (查询)  = 你想找什么书                 │
│     "我想找一本关于 AI 的书"                     │
│                                                 │
│  🏷️ Key (键)     = 书架上的标签                  │
│     "计算机"、"人工智能"、"编程"                 │
│                                                 │
│  📖 Value (值)   = 书的内容                      │
│     真正的书本内容                               │
│                                                 │
│  匹配过程：                                      │
│  Query 和 Key 比较 → 得到注意力分数              │
│  高分数的 Key 对应的 Value 更重要                │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 1.4 注意力计算过程

**步骤图解：**

```
输入句子：["我", "喜欢", "苹果"]

第1步：生成 Q、K、V
───────────────────────────
"我"    → Q₁, K₁, V₁
"喜欢"  → Q₂, K₂, V₂
"苹果"  → Q₃, K₃, V₃

第2步：计算注意力分数
───────────────────────────
Q₁ × K₁ = 高 (自己关注自己)
Q₁ × K₂ = 中 (有点关系)
Q₁ × K₃ = 低 (关系不大)

第3步：Softmax 归一化
───────────────────────────
把分数变成概率（加起来等于1）
[0.6, 0.3, 0.1]

第4步：加权求和
───────────────────────────
输出 = 0.6×V₁ + 0.3×V₂ + 0.1×V₃
```

### 1.5 自注意力 vs 交叉注意力

```
┌─────────────────┬─────────────────────────────┐
│     类型        │           说明              │
├─────────────────┼─────────────────────────────┤
│   自注意力       │ 同一句话内部词与词的关系     │
│  Self-Attention │ "我喜欢苹果"中"我"和"喜欢"   │
├─────────────────┼─────────────────────────────┤
│   交叉注意力     │ 两句话之间词的关系          │
│Cross-Attention  │ 原文和译文之间的关系         │
└─────────────────┴─────────────────────────────┘
```

## 📖 第二部分：Transformer 架构

### 2.1 Transformer 的诞生

**2017年的革命：**

Google 发表论文《Attention Is All You Need》，提出了 Transformer 架构。

```
之前：RNN、LSTM（像读书一样，从左到右依次处理）
问题：太慢，难以并行，长句子会"忘记"开头

Transformer：
  ✅ 可以并行处理（同时看所有词）
  ✅ 长距离关系也能捕捉
  ✅ 训练更快，效果更好
```

### 2.2 Transformer 结构图解

```
┌─────────────────────────────────────────────┐
│                  Transformer                │
├─────────────────────────────────────────────┤
│                                             │
│  输出嵌入 ←─────────────────────────────┐   │
│      ↓                                   │   │
│  ┌─────────────┐                        │   │
│  │   解码器    │ ← 来自编码器的注意力    │   │
│  │  (Decoder)  │                        │   │
│  └─────────────┘                        │   │
│      ↓                                   │   │
│  ┌─────────────┐                        │   │
│  │   编码器    │ ──────────────────────┘   │
│  │  (Encoder)  │                           │
│  └─────────────┘                           │
│      ↓                                     │
│  输入嵌入 + 位置编码                        │
│                                             │
└─────────────────────────────────────────────┘
```

### 2.3 核心组件详解

**1. 输入嵌入（Embedding）**

```
把文字变成数字

"猫" → [0.1, 0.3, -0.2, ...]
"狗" → [0.15, 0.28, -0.1, ...]

相似的词，数字也相似！
```

**2. 位置编码（Positional Encoding）**

```
问题：Transformer 同时看所有词，不知道顺序

解决：给每个位置加上"位置标签"

位置1: [sin(1), cos(1), sin(1/100), ...]
位置2: [sin(2), cos(2), sin(2/100), ...]
位置3: [sin(3), cos(3), sin(3/100), ...]

这样模型就知道"猫在狗前面"还是"狗在猫前面"了！
```

**3. 多头注意力（Multi-Head Attention）**

```
为什么需要多个"头"？

就像人从多个角度观察事物：

头1：关注语法关系（主语-谓语）
头2：关注语义关系（同义词）
头3：关注指代关系（他→某人）
...

最后把多个头的结果合并！
```

**4. 前馈网络（Feed Forward）**

```
进一步处理注意力结果

注意力 → 找关系
前馈网络 → 深度理解

两者配合，效果更强！
```

## 📖 第三部分：三种主流架构

### 3.1 编码器-解码器架构

```
┌─────────────┐     ┌─────────────┐
│   编码器    │ ──→ │   解码器    │
│  Encoder    │     │  Decoder    │
└─────────────┘     └─────────────┘
     ↓                    ↓
   理解输入            生成输出

例子：翻译
编码器：理解中文句子
解码器：生成英文句子

代表模型：T5、BART
适合任务：翻译、摘要
```

### 3.2 因果解码器架构

```
┌─────────────────────────────────┐
│           解码器                 │
│         Decoder Only            │
│                                 │
│  只能看到之前的内容：            │
│  预测"猫"时，看不到"狗"         │
│                                 │
└─────────────────────────────────┘

例子：写作文
看前面的字，预测下一个字
像人类写作一样！

代表模型：GPT 系列、DeepSeek、LLaMA
适合任务：对话、写作、代码生成
```

### 3.3 前缀解码器架构

```
┌─────────────────────────────────┐
│      Prefix Decoder             │
├─────────────────────────────────┤
│  前缀部分：双向看（编码器模式）  │
│  生成部分：单向看（解码器模式）  │
└─────────────────────────────────┘

例子：问答
问题部分：可以前后看
回答部分：只能看前面

代表模型：GLM
适合任务：理解和生成兼顾
```

### 3.4 架构对比总结

```
┌─────────────┬───────────────┬───────────────┬───────────────┐
│   架构      │    编码器     │    解码器     │   前缀解码器  │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 代表模型    │ BERT          │ GPT           │ GLM           │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 擅长任务    │ 理解          │ 生成          │ 理解+生成     │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 看文本方式  │ 双向看        │ 单向看        │ 混合          │
├─────────────┼───────────────┼───────────────┼───────────────┤
│ 典型应用    │ 分类、问答    │ 写作、对话    │ 通用任务      │
└─────────────┴───────────────┴───────────────┴───────────────┘
```
