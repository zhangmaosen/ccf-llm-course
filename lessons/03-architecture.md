## 📖 课程内容

### 一、注意力机制

#### 生活中的注意力

上课时，你会注意什么？
- 👀 老师的讲解
- 📝 黑板上的板书
- 🔔 同学的提问

人类的注意力是**有选择性**的，AI 也可以！

#### 注意力机制原理

**核心概念：Query、Key、Value**

想象你在图书馆找书：
- **Query（查询）**：你想找的主题
- **Key（键）**：每本书的标签
- **Value（值）**：书的内容

通过计算 Query 和 Key 的相似度，决定关注哪些 Value。

#### Softmax 函数

将数值转换为概率分布：
```
输入：[2, 1, 0.1]
输出：[0.659, 0.242, 0.099]
```
所有输出加起来等于 1，表示"注意力分配"。

### 二、Transformer 架构

#### 2017 年的革命

Google 发表的论文《Attention Is All You Need》，提出了 Transformer 架构，改变了整个 AI 领域。

#### Transformer 的组成

```
┌─────────────────────────────┐
│         输出层              │
├─────────────────────────────┤
│  解码器 (Decoder) × N层     │
├─────────────────────────────┤
│  编码器 (Encoder) × N层     │
├─────────────────────────────┤
│         输入层              │
│   (词嵌入 + 位置编码)        │
└─────────────────────────────┘
```

**核心组件**：
1. **输入编码**：将文字转换为数字表示
2. **位置编码**：告诉模型词语的顺序
3. **多头注意力**：从多个角度关注信息
4. **前馈网络**：进一步处理信息

---

### 三、主流架构类型

#### 1. 编码器-解码器架构

```
编码器 → 理解输入
解码器 → 生成输出
```

**代表模型**：T5、BART
**应用**：翻译、摘要

#### 2. 因果解码器架构

```
只能看到之前的内容，逐个生成
```

**代表模型**：GPT 系列、DeepSeek
**应用**：对话、写作

#### 3. 前缀解码器架构

```
编码部分可以双向看，解码部分单向
```

**代表模型**：GLM
**应用**：理解和生成兼顾

### 实验 2：架构对比

| 任务 | 推荐架构 | 原因 |
|-----|---------|------|
| 文本翻译 | 编码器-解码器 | 需要理解和生成 |
| 写作文 | 因果解码器 | 逐字生成 |
| 阅读理解 | 编码器 | 只需要理解 |
