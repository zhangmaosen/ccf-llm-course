## 📖 第一部分：什么是预训练？

### 1.1 类比理解

**传统方法 vs 预训练方法：**

```
┌─────────────────────────────────────────────────┐
│               传统方法（从头训练）               │
├─────────────────────────────────────────────────┤
│  就像：每个任务都从零开始培养一个学生           │
│  问题：需要大量标注数据，成本高                 │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│               预训练方法（迁移学习）             │
├─────────────────────────────────────────────────┤
│  就像：先上通识课，再上专业课                   │
│  优势：先学通用知识，再针对任务微调             │
└─────────────────────────────────────────────────┘
```

### 1.2 两阶段训练

```
第一阶段：预训练（通识教育）
─────────────────────────────
任务：阅读海量文本，学习语言规律
数据：互联网上的文本（无标签）
目标：预测下一个词

第二阶段：微调（专业培训）
─────────────────────────────
任务：针对具体任务优化
数据：标注好的任务数据
目标：完成特定任务（翻译、问答等）
```

### 1.3 预训练的规模

**GPT-3 的预训练数据：**

```
┌───────────────────────────────────────┐
│           GPT-3 训练数据              │
├───────────────────────────────────────┤
│ 📚 书籍          几百万本             │
│ 🌐 网页          几十亿页             │
│ 📝 论文          几百万篇             │
│ 💬 对话          几十亿条             │
│ 📰 新闻          几千万篇             │
├───────────────────────────────────────┤
│ 总词数          约 5000 亿词          │
│ 相当于          每天读一本书          │
│                读 1.4 亿年！          │
└───────────────────────────────────────┘
```

## 📖 第二部分：预训练任务

### 🎬 视频导学
📺 [GPT 预训练原理](https://www.bilibili.com/video/BV1xxx)
🔗 [语言模型在线演示](https://huggingface.co/spaces/bigscience/bloom)

### 2.1 下一个词元预测

**最经典的预训练任务：**

```
游戏：猜下一个词

输入：今天天气真___
可能答案：好、不错、糟糕、热...

模型学习：
- 统计规律："天气真"后面常接"好"
- 语义理解：根据上下文判断

这是 GPT 系列的核心任务！
```

**为什么有效？**

```
要准确预测下一个词，模型必须学会：
✅ 语法规则（句子要通顺）
✅ 语义理解（意思要对）
✅ 世界知识（巴黎是法国首都）
✅ 上下文关系（代词指代谁）
```

### 2.2 掩码语言建模（MLM）

**BERT 的任务：填空题**

```
输入：我 [MASK] 吃 苹果
目标：预测 [MASK] 是什么

可能的答案：喜欢、爱吃、要...

BERT 会把句子中 15% 的词遮住
让模型猜测被遮住的词
```

**对比两种任务：**

```
┌─────────────┬─────────────────┬─────────────────┐
│    任务     │     GPT 方式    │    BERT 方式    │
├─────────────┼─────────────────┼─────────────────┤
│   预测方式  │  预测下一个词   │  预测被遮住的词 │
├─────────────┼─────────────────┼─────────────────┤
│   看的方向  │  只能看左边     │  左右都能看     │
├─────────────┼─────────────────┼─────────────────┤
│   擅长任务  │  生成文本       │  理解文本       │
├─────────────┼─────────────────┼─────────────────┤
│   应用场景  │  写作、对话     │  分类、问答     │
└─────────────┴─────────────────┴─────────────────┘
```

### 2.3 其他预训练任务

**1. 下 N 个词预测**

```
不只是预测下一个，而是预测接下来 N 个词

输入：今天天气
预测：真好，我们去公园玩吧！
```

**2. 句子顺序预测**

```
判断两个句子是否连续

句子A：今天天气真好
句子B：我们去公园吧
答案：是连续的 ✅

句子A：今天天气真好
句子B：苹果是红色的
答案：不连续 ❌
```

## 📖 第三部分：自监督学习

### 3.1 什么是自监督学习？

**三种学习方式对比：**

```
┌─────────────────────────────────────────────────┐
│                 监督学习                         │
├─────────────────────────────────────────────────┤
│ 需要人工标注答案                                 │
│ 例子：图片分类，每张图都要标"猫"或"狗"          │
│ 成本：高 💰💰💰                                 │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│                 无监督学习                       │
├─────────────────────────────────────────────────┤
│ 没有任何标签，自己找规律                         │
│ 例子：聚类，把相似的物品分组                     │
│ 难度：学习目标不明确                            │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│                 自监督学习                       │
├─────────────────────────────────────────────────┤
│ 数据本身提供监督信号！                           │
│ 例子：预测下一个词，答案就在文本里               │
│ 成本：低 💰，效果：好 ✅                        │
└─────────────────────────────────────────────────┘
```

### 3.2 自监督的妙处

```
文本本身就是"带答案"的！

原始文本："我 喜欢 吃 苹果"
────────────────────────────
输入："我 喜欢 吃"
答案："苹果"（就在原文中！）

不需要人工标注，数据量可以无限大！
```

### 3.3 自监督学习的类型

```
┌─────────────────┬─────────────────────────────┐
│      类型       │            例子              │
├─────────────────┼─────────────────────────────┤
│  对比学习       │ 区分相似和不相似的样本       │
├─────────────────┼─────────────────────────────┤
│  掩码预测       │ 猜被遮住的内容（BERT）       │
├─────────────────┼─────────────────────────────┤
│  自回归         │ 预测下一个词（GPT）          │
├─────────────────┼─────────────────────────────┤
│  生成式         │ 重建原始数据                 │
└─────────────────┴─────────────────────────────┘
```

## 📖 第四部分：训练的关键要素

### 4.1 Batch Size（批次大小）

**类比：做作业**

```
Batch Size = 1：一次做一道题
  优点：每道题都仔细思考
  缺点：太慢，效率低

Batch Size = 32：一次做32道题
  优点：可以对比学习，效率高
  缺点：需要更多"脑子"（内存）

Batch Size = 1000：一次做1000道题
  优点：更快
  缺点：可能"消化不良"，效果变差
```

### 4.2 学习率（Learning Rate）

**类比：调节步子大小**

```
┌─────────────────────────────────────────┐
│           寻找山顶（最优解）             │
├─────────────────────────────────────────┤
│                                         │
│  学习率大 = 步子大                       │
│    ✅ 走得快                            │
│    ❌ 可能跳过山顶                      │
│                                         │
│  学习率小 = 步子小                       │
│    ✅ 走得稳                            │
│    ❌ 走得慢                            │
│                                         │
│  好的策略：                             │
│    开始大步走，接近山顶时小步走          │
│    这叫"学习率衰减"                     │
│                                         │
└─────────────────────────────────────────┘
```

### 4.3 优化器

**常用优化器：**

```
┌─────────────┬───────────────────────────────┐
│   优化器    │              特点              │
├─────────────┼───────────────────────────────┤
│    SGD      │  最简单，像盲人下山            │
│             │  只知道当前坡度                │
├─────────────┼───────────────────────────────┤
│    Adam     │  最常用，像有记忆的登山者      │
│             │  记住之前的方向，调整步伐      │
├─────────────┼───────────────────────────────┤
│   AdamW     │  Adam 的改进版                 │
│             │  更稳定，效果更好              │
└─────────────┴───────────────────────────────┘
```
