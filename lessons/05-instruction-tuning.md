## 🚀 课前思考

**预训练模型的"问题"：**

```
用户：请帮我写一首诗
预训练模型：关于春天的诗，描写了温暖的阳光和盛开的花朵...
            （它把这句话当成了要续写的内容！）

期望的回答：
预训练模型：（应该开始写诗！）

问题：模型不知道这是"指令"，以为要续写文本
```

**解决方法：指令微调！**

让模型学会"这是人类给我的任务，我需要完成它"。

## 📖 第一部分：为什么需要指令微调？

### 1.1 预训练模型的局限

```
┌─────────────────────────────────────────────────┐
│            预训练模型的能力                      │
├─────────────────────────────────────────────────┤
│ ✅ 能预测下一个词                               │
│ ✅ 学会了语言的规律                             │
│ ❌ 不知道什么是"指令"                          │
│ ❌ 不知道要"完成任务"                          │
│ ❌ 回答可能不符合人类期望                       │
└─────────────────────────────────────────────────┘
```

### 1.2 预训练 vs 微调

```
预训练（通识教育）
──────────────────────
目标：学习语言规律
任务：预测下一个词
数据：海量无标签文本
能力：语言建模

微调（专业培训）
──────────────────────
目标：学会遵循指令
任务：按要求回答
数据：指令-回答对
能力：任务完成
```

### 1.3 指令微调的效果

**微调前后对比：**

```
┌─────────────────────────────────────────────────┐
│ 用户：请把这句话翻译成英语：今天天气真好        │
├─────────────────────────────────────────────────┤
│ 微调前：今天天气真好，我们决定去公园散步...     │
│        （继续写句子）                           │
├─────────────────────────────────────────────────┤
│ 微调后：The weather is really nice today.       │
│        （按要求翻译！）                         │
└─────────────────────────────────────────────────┘
```

## 📖 第二部分：指令数据的构成

### 2.1 指令数据的三要素

```
┌─────────────────────────────────────────────────┐
│              指令数据结构                        │
├─────────────────────────────────────────────────┤
│                                                 │
│  📋 Instruction（指令）                         │
│     告诉模型要做什么                            │
│     例："请将以下句子翻译成英语"                │
│                                                 │
│  📥 Input（输入）                               │
│     任务的具体内容                              │
│     例："今天天气真好"                          │
│                                                 │
│  📤 Output（输出）                              │
│     期望的答案                                  │
│     例："The weather is nice today."           │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 2.2 指令数据示例

**示例 1：翻译任务**
```json
{
  "instruction": "请将以下中文翻译成英语",
  "input": "我喜欢学习人工智能",
  "output": "I like learning about artificial intelligence."
}
```

**示例 2：问答任务**
```json
{
  "instruction": "回答以下问题",
  "input": "中国的首都是哪里？",
  "output": "中国的首都是北京。"
}
```

**示例 3：写作任务**
```json
{
  "instruction": "请写一首关于春天的诗，四行",
  "input": "",
  "output": "春风轻抚绿柳枝，\n花开满园香满池。\n燕舞莺歌迎暖日，\n人间三月最相思。"
}
```

## 📖 第三部分：指令数据集构建

### 3.1 数据来源

```
┌─────────────────┬───────────────────────────────┐
│     来源        │              说明              │
├─────────────────┼───────────────────────────────┤
│  人工编写       │ 专家设计高质量指令             │
│                 │ 质量高，但成本高 💰💰💰       │
├─────────────────┼───────────────────────────────┤
│  自动生成       │ 用大模型生成指令数据           │
│                 │ 量大，但质量参差不齐           │
├─────────────────┼───────────────────────────────┤
│  公开数据集     │ 利用已有的问答数据             │
│                 │ 成本低，但需要转换格式         │
├─────────────────┼───────────────────────────────┤
│  用户反馈       │ 收集真实用户的问题和回答       │
│                 │ 最贴近实际使用场景             │
└─────────────────┴───────────────────────────────┘
```

### 3.2 高质量指令的标准

```
好的指令数据应该：

✅ 指令清晰明确
   "翻译成英语" vs "处理一下这个"

✅ 覆盖多样任务
   翻译、问答、写作、编程...

✅ 答案准确有用
   经过人工审核或高质量模型生成

✅ 难度梯度合理
   简单任务 + 复杂任务
```

### 3.3 指令数据量

```
常见指令数据集规模：

┌─────────────────┬───────────────┐
│     数据集      │    数量       │
├─────────────────┼───────────────┤
│ Alpaca          │ 5.2 万条      │
│ Dolly           │ 1.5 万条      │
│ OpenAssistant   │ 16 万条       │
│ ShareGPT        │ 9 万条        │
└─────────────────┴───────────────┘

研究表明：几万条高质量指令就足够！
```

## 📖 第四部分：参数高效微调

### 🎬 视频导学
📺 [LoRA 原理详解](https://www.bilibili.com/video/BV1xxx)
📄 [LoRA 论文图解](https://arxiv.org/abs/2106.09685)

### 4.1 问题：全量微调太贵

```
全量微调 = 更新所有参数

GPT-3 1750亿参数：
- 需要巨大的 GPU 内存
- 训练成本非常高
- 每个任务都要存一份模型

问题：普通用户根本玩不起！
```

### 4.2 解决：LoRA

**LoRA（Low-Rank Adaptation）的巧妙想法：**

```
┌─────────────────────────────────────────────────┐
│                  LoRA 原理                       │
├─────────────────────────────────────────────────┤
│                                                 │
│  原来：直接修改所有参数                          │
│  参数量：1750 亿                                │
│                                                 │
│  LoRA：只训练少量"适配器"参数                   │
│  参数量：可能是原来的 1% 或更少                 │
│                                                 │
│  类比：                                          │
│  全量微调 = 重新装修整个房子                    │
│  LoRA = 只换几个关键位置的家具                  │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 4.3 其他参数高效方法

```
┌─────────────────┬───────────────────────────────┐
│      方法       │              说明              │
├─────────────────┼───────────────────────────────┤
│  Adapter        │ 在模型中插入小模块            │
├─────────────────┼───────────────────────────────┤
│  Prefix Tuning  │ 在输入前加可学习的"前缀"      │
├─────────────────┼───────────────────────────────┤
│  Prompt Tuning  │ 优化提示词的表示              │
├─────────────────┼───────────────────────────────┤
│  LoRA           │ 低秩分解，最常用              │
└─────────────────┴───────────────────────────────┘
```
