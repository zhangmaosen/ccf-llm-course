# 第五课：指令微调

> 让模型听懂人类指令

## 🎯 学习目标

- 理解指令微调的概念
- 了解指令数据的构成
- 认识微调的作用

---

## 📖 课程内容

### 一、预训练 vs 微调

#### 预训练模型的能力

预训练后的模型：
- ✅ 能够预测下一个词
- ✅ 学会了语言的规律
- ❌ 不一定能理解人类指令

#### 为什么需要微调？

**例子**：
```
用户：请帮我写一首关于春天的诗
预训练模型：可能继续这个句子，而不是写诗
微调后模型：理解这是指令，开始写诗
```

---

### 二、什么是指令微调？

#### 定义

**指令微调**：用"指令-回答"格式的数据训练模型，让模型学会遵循人类指令。

#### 指令数据的构成

```
{
  "instruction": "请将以下句子翻译成英语",
  "input": "今天天气很好",
  "output": "The weather is nice today."
}
```

**三要素**：
1. **Instruction（指令）**：告诉模型做什么
2. **Input（输入）**：任务的具体内容
3. **Output（输出）**：期望的答案

---

### 三、指令数据集构建

#### 数据来源

1. **人工编写**：专家设计高质量指令
2. **自动生成**：用大模型生成指令数据
3. **公开数据集**：利用已有的问答数据

#### 数据质量很重要

- 📌 指令清晰明确
- 📌 答案准确有用
- 📌 任务多样化

---

### 四、微调的效果

#### 微调前后的对比

| 能力 | 微调前 | 微调后 |
|-----|-------|-------|
| 理解指令 | 差 | 好 |
| 回答质量 | 不稳定 | 稳定 |
| 任务泛化 | 有限 | 较强 |

#### 指令遵循能力

模型学会：
- 🎯 识别指令意图
- 🎯 按要求完成任务
- 🎯 给出合适格式
