## 🚀 课前思考

**如果 AI 不加约束，会发生什么？**

```
用户：如何制作炸弹？
AI（未对齐）：好的，我来教你材料和方法...

用户：帮我写一篇论文
AI（未对齐）：好的，这是一篇抄袭的论文...

用户：这个产品怎么样？
AI（未对齐）：这是最好的产品！（实际上根本不存在）
```

**这些问题很危险！** 所以需要对 AI 进行"对齐"。

### 1.1 AI 的潜在风险

```
┌─────────────────────────────────────────────────┐
│              AI 可能产生的问题                   │
├─────────────────────────────────────────────────┤
│                                                 │
│  🔴 有害内容                                    │
│     生成暴力、歧视、违法内容                    │
│                                                 │
│  🟠 虚假信息                                    │
│     编造不存在的事实、错误信息                  │
│                                                 │
│  🟡 无用回答                                    │
│     回答偏离主题、没有帮助                      │
│                                                 │

│     反映训练数据中的社会偏见                    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 1.2 对齐的目标

```
对齐 = 让 AI 的行为符合人类期望

就像教育孩子：
- 要有礼貌（语言得体）
- 要诚实（不说谎）
- 要有帮助（解决问题）
- 要有道德（不做坏事）
```

## 📖 第二部分：对齐的三个标准

### 2.1 无害性（Harmless）

**AI 不应该：**

```
❌ 生成有害、危险的内容
❌ 帮助进行违法活动
❌ 传播歧视和仇恨
❌ 提供危险建议
```

**对齐前后的对比：**

```
┌─────────────────────────────────────────────────┐
│ 用户：如何制作炸弹？                            │
├─────────────────────────────────────────────────┤
│ 对齐前：好的，我来教你...                       │
│        （提供详细步骤）                         │
├─────────────────────────────────────────────────┤
│ 对齐后：抱歉，我不能提供制作危险物品的信息。   │
│        如果你对化学实验感兴趣，我可以推荐一些   │
│        安全的科学实验。                        │
└─────────────────────────────────────────────────┘
```

### 2.2 有用性（Helpful）

**AI 应该：**

```
✅ 提供有价值的帮助
✅ 准确回答问题
✅ 高效完成任务
✅ 主动澄清需求
```

**有用 vs 无用的回答：**

```
用户：如何学习编程？

无用的回答：
"编程很重要，你可以去学。"

有用的回答：
"学习编程可以从以下几步开始：

1. 选择语言：推荐 Python，语法简单
2. 学习资源：
   - 官方教程（免费）
   - B站视频课程
   - 在线练习平台
3. 实践项目：从小程序开始
4. 加入社区：和其他学习者交流

需要我推荐具体的学习资源吗？"
```

### 2.3 诚实性（Honest）

**AI 应该：**

```
✅ 说真话
✅ 承认不确定
✅ 不编造信息
✅ 区分事实和观点
```

**诚实的表现：**

```
用户：爱因斯坦是在哪一年获得诺贝尔奖的？

诚实的回答：
"爱因斯坦在1921年获得诺贝尔物理学奖。"

承认不确定：
"我不太确定这个信息的准确性，建议你查证一下。"
"据我所知，这个信息可能有误..."

拒绝编造：
"抱歉，我没有找到关于这个问题的确切信息。"
```

## 📖 第三部分：对齐的方法

### 3.1 RLHF（基于人类反馈的强化学习）

```
RLHF 三个步骤：

第1步：收集人类反馈
───────────────────────
给模型的不同回答排序：
回答A > 回答B > 回答C

第2步：训练奖励模型
───────────────────────
让 AI 学会判断哪个回答更好

第3步：强化学习优化
───────────────────────
用奖励模型指导 AI 生成更好的回答
```

### 3.2 DPO（直接偏好优化）

```
DPO：更简单的对齐方法

不需要训练单独的奖励模型
直接用人类偏好数据优化模型

优点：
✅ 更简单
✅ 更稳定
✅ 效果好
```

### 3.3 对齐方法的对比

```
┌─────────────┬─────────────────┬─────────────────┐
│    方法     │      优点       │      缺点       │
├─────────────┼─────────────────┼─────────────────┤
│   RLHF      │ 效果好，成熟    │ 复杂，不稳定    │
├─────────────┼─────────────────┼─────────────────┤
│    DPO      │ 简单，稳定      │ 适用场景有限    │
├─────────────┼─────────────────┼─────────────────┤
│  安全训练   │ 减少有害内容    │ 可能过度限制    │
└─────────────┴─────────────────┴─────────────────┘
```

## 📖 第四部分：幻象问题

### 4.1 什么是幻象？

**幻象（Hallucination）：** AI 编造不存在的信息

```
例子：

用户：介绍一下《时间的河流》这本书
AI：这本书由张三所著，出版于2020年，
    讲述了一个关于时间旅行的故事...

问题：这本书根本不存在！AI 编造了所有内容！
```

### 4.2 幻象的类型

```
┌─────────────────┬───────────────────────────────┐
│      类型       │              例子              │
├─────────────────┼───────────────────────────────┤
│  事实性幻象     │ "爱因斯坦出生于1990年"        │
│                 │ （事实错误）                   │
├─────────────────┼───────────────────────────────┤
│  虚构内容       │ 编造不存在的人、书、事件       │
├─────────────────┼───────────────────────────────┤
│  逻辑矛盾       │ 前后回答不一致                 │
├─────────────────┼───────────────────────────────┤
│  过度自信       │ 对错误信息非常确定             │
└─────────────────┴───────────────────────────────┘
```

### 4.3 幻象的成因

```
原因 1：训练数据有错误
─────────────────────
网上的信息本身就有错误

原因 2：模型过度"创造"
─────────────────────
为了回答问题，编造内容

原因 3：不确定时仍回答
─────────────────────
不知道也说，不承认"我不知道"

原因 4：问题诱导
─────────────────────
用户的问题暗示了一个错误前提
```

### 4.4 如何缓解幻象？

```
方法 1：让 AI 学会说"不知道"
─────────────────────────────
不确定的信息，坦诚说明

方法 2：引用来源
─────────────────────────────
回答时提供信息来源

方法 3：多模型验证
─────────────────────────────
用多个模型交叉验证

方法 4：用户自己验证
─────────────────────────────
重要信息要自行查证！
```

```
用户：帮我写一篇关于人工智能的论文

AI：好的，这是一篇完整的论文：
    [一篇抄袭的论文]

```

用户：这个手机多少钱？

AI：这款手机售价 2999 元。

问题：如果这款手机实际上不存在，这是什么的幻象？
