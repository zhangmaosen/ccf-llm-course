## 🚀 课前思考

**模型输出的是什么？**

```
输入："今天天气"
模型输出：一个概率分布

{好: 0.60, 不错: 0.20, 糟糕: 0.15, 热: 0.05}

问题：如何从这个概率分布中选择一个词？
这就是"解码"要解决的问题！
```

## 📖 第一部分：解码方法

### 1.1 从概率到文字

```
模型输出的不是文字，而是概率

例如：
输入："我喜欢"
模型输出：
{吃: 0.30, 看: 0.25, 学习: 0.20, 运动: 0.15, 睡觉: 0.10}

解码就是：从这些概率中选择一个词
```

### 1.2 贪心搜索（Greedy Search）

```
方法：每次选概率最高的词

输入："我喜欢"
输出概率：{吃: 0.30, 看: 0.25, 学习: 0.20}
选择："吃"（最高）

继续："我喜欢吃"
输出概率：{苹果: 0.50, 香蕉: 0.30, 橘子: 0.20}
选择："苹果"

结果："我喜欢吃苹果"

✅ 优点：速度快
❌ 缺点：可能错过更好的答案
```

### 1.3 束搜索（Beam Search）

```
方法：保留多个候选，最后选最好的

Beam Width = 2（保留 2 个候选）

第1步：
候选1: "我喜欢吃" (概率 0.30)
候选2: "我喜欢看" (概率 0.25)

第2步：
候选1: "我喜欢吃苹果" (概率 0.30×0.50 = 0.15)
候选2: "我喜欢看书" (概率 0.25×0.40 = 0.10)

✅ 优点：质量更好
❌ 缺点：计算量大
```

### 1.4 采样方法

**温度采样（Temperature Sampling）**

```
温度 = 1：正常分布
温度 < 1：更确定（概率高的更高）
温度 > 1：更随机（概率趋于平均）

温度 = 0.5（更确定）:
{吃: 0.70, 看: 0.20, 学习: 0.10}
更可能选"吃"

温度 = 1.5（更随机）:
{吃: 0.40, 看: 0.30, 学习: 0.30}
选择更均匀
```

**Top-k 和 Top-p 采样**

```
Top-k：只从前 k 个词中选
Top-p：只从累计概率达到 p 的词中选

这样既保证多样性，又避免选到太离谱的词
```

## 📖 第三部分：推理加速

### 3.1 为什么需要加速？

```
问题：大模型推理很慢！

GPT-3 回答一个问题：
- 需要数千亿次计算
- 可能需要几秒钟
- 用户等待体验差

解决：推理加速技术
```

### 3.2 加速技术

**KV Cache（键值缓存）**

```
问题：每次生成都要重新计算之前的结果

解决：把之前计算结果存起来，复用！

不用 KV Cache：每次都要算全部
使用 KV Cache：只算新生成的部分

提速效果：几倍到几十倍！
```

**推测解码（Speculative Decoding）**

```
思路：用小模型猜，大模型验证

第1步：小模型快速猜测多个词
第2步：大模型验证哪些是对的
第3步：一次性接受多个词

效果：速度快，质量不变
```

**Flash Attention**

```
优化注意力计算的内存访问
减少 GPU 内存读写次数

效果：速度提升 2-4 倍
内存占用减少一半
```

### 4.2 模型压缩

**量化（Quantization）**

```
降低模型精度，减小体积

FP16（16位浮点）→ INT8（8位整数）
体积减半，速度更快，精度略降

FP32 → FP16 → INT8 → INT4
精度降低，体积减小
```

**蒸馏（Distillation）**

```
让小模型学习大模型的知识

大模型（老师）→ 小模型（学生）

效果：模型变小，能力保留大部分
```

**剪枝（Pruning）**

```
删除不重要的参数

模型参数很多，有些"不重要"
删掉这些参数，模型更小更快
```
