## 🚀 课前思考

**同样的 AI，为什么回答质量差这么多？**

```
❌ 差的提示：
"写作文"

回答：（可能写任何主题，质量不稳定）

✅ 好的提示：
"请写一篇关于'春天的校园'的作文，500字左右，
使用描写和抒情的表达方式，要有具体的景物描写。"

回答：（高质量、符合要求的内容）
```

**关键：** 提示词的质量决定了 AI 输出的质量！

## 📖 第一部分：什么是提示工程？

### 1.1 提示的作用

```
┌─────────────────────────────────────────────────┐
│                  人机对话                        │
├─────────────────────────────────────────────────┤
│                                                 │
│  你 ──→ 提示（Prompt）──→ AI 模型 ──→ 回答     │
│                                                 │
│  提示 = 你给 AI 的指令 + 上下文                 │
│                                                 │
│  好提示 = 清晰 + 具体 + 有上下文                │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 1.2 提示学习 vs 微调

```
┌─────────────────┬───────────────────────────────┐
│      方法       │              说明              │
├─────────────────┼───────────────────────────────┤
│    微调         │ 修改模型参数                  │
│                 │ 需要训练，成本高              │
├─────────────────┼───────────────────────────────┤
│   提示学习      │ 不修改模型，设计好的提示      │
│                 │ 即刻生效，成本低              │
└─────────────────┴───────────────────────────────┘

类比：
微调 = 给 AI 上课培训
提示学习 = 给 AI 清晰的指令
```

### 1.3 提示学习的局限

```
提示学习不能：

❌ 给 AI 新知识
   （AI 只能用已有的知识）

❌ 让 AI 记住长内容
   （上下文长度有限）

❌ 解决超出能力的问题
   （AI 能力有上限）
```

## 📖 第二部分：提示设计技巧

### 2.1 技巧一：明确任务

```
❌ 模糊的提示：
"分析一下"

✅ 明确的提示：
"请分析这篇文章的主要观点，并列出三个支持的论据"
```

**明确任务的要素：**

```
好的提示包含：

1. 动作词：总结、翻译、分析、解释、写...
2. 对象：要处理的内容
3. 要求：格式、长度、风格
4. 限制：必须包含/排除的内容
```

### 2.2 技巧二：提供上下文

```
❌ 没有上下文：
"翻译：apple"

（不知道要翻译成什么语言）

✅ 有上下文：
"请把以下英文单词翻译成中文：apple"

✅ 更多上下文：
"我是一名小学生，请把 'apple' 这个单词翻译成中文，
并给我一个简单的造句例子。"
```

### 2.3 技巧三：给出示例

**Few-shot 提示（少样本学习）：**

```
请判断以下句子的情感倾向：

示例 1：
句子：今天天气真好！
情感：积极

示例 2：
句子：我很失望。
情感：消极

请判断：
句子：这部电影太精彩了！
情感：
```

**示例的力量：**

```
┌─────────────────────────────────────────────────┐
│  Zero-shot（零样本）：不给示例                   │
│  可能准确率：70%                                │
├─────────────────────────────────────────────────┤
│  Few-shot（少样本）：给几个示例                  │
│  可能准确率：90%                                │
└─────────────────────────────────────────────────┘
```

### 2.4 技巧四：角色扮演

```
普通提示：
"解释一下什么是机器学习"

角色扮演提示：
"你是一位小学科学老师，请用简单易懂的方式，
给一个10岁的孩子解释什么是机器学习，
可以举生活中的例子。"
```

**常用角色：**

```
┌─────────────────┬───────────────────────────────┐
│      角色       │              效果              │
├─────────────────┼───────────────────────────────┤
│  老师           │ 解释更清晰，用简单语言        │
├─────────────────┼───────────────────────────────┤
│  专家           │ 回答更专业，深入              │
├─────────────────┼───────────────────────────────┤
│  朋友           │ 语气更亲切，有互动            │
├─────────────────┼───────────────────────────────┤
│  批评家         │ 指出问题，提出不同观点        │
└─────────────────┴───────────────────────────────┘
```

### 2.5 技巧五：分步骤

```
请按以下步骤帮我分析这篇文章：

第1步：用一句话总结文章主旨
第2步：列出文章的三个关键观点
第3步：指出文章的一个可能的不足
第4步：给出你的改进建议

文章内容：...
```

### 2.6 技巧六：指定格式

```
请用以下格式回答：

## 概念定义
（什么是...）

## 主要应用
1. ...
2. ...
3. ...

## 总结
（一句话总结）

问题：什么是人工智能？
```

## 📖 第三部分：思维链提示

### 🎬 视频导学
📺 [思维链原理与应用](https://www.bilibili.com/video/BV1xxx)
📄 [Chain of Thought 论文](https://arxiv.org/abs/2201.11903)

### 3.1 什么是思维链？

**Chain of Thought（CoT）：让 AI "一步步思考"**

```
❌ 直接回答：
问题：小明有5个苹果，给了小红2个，又买了3个，现在有几个？
回答：6个

✅ 思维链回答：
问题：小明有5个苹果，给了小红2个，又买了3个，现在有几个？
回答：
让我们一步步思考：
1. 小明原来有 5 个苹果
2. 给了小红 2 个，还剩 5 - 2 = 3 个
3. 又买了 3 个，现在有 3 + 3 = 6 个
所以答案是 6 个
```

### 3.2 思维链的触发方式

```
方法 1：直接要求
"请一步步思考并回答"

方法 2：给出示例
在提示中给出一个带思考过程的示例

方法 3：Zero-shot CoT
只需要加上："Let's think step by step"
```

### 3.3 思维链的适用场景

```
✅ 适合思维链：
- 数学计算
- 逻辑推理
- 复杂问题分解
- 代码编写

❌ 不需要思维链：
- 简单问答
- 翻译
- 摘要
```

## 📖 第四部分：使用 API 与大模型交互

### 4.1 API 是什么？

```
API（应用程序接口）= 与 AI 模型对话的"桥梁"

你的程序 ──→ API 请求 ──→ AI 模型
              ↑                    │
              └────── API 响应 ←───┘
```

### 4.2 基本调用示例

```python
# 伪代码示例
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "user", "content": "你好，请介绍一下自己"}
    ]
)

print(response.choices[0].message.content)
```

### 4.3 重要参数

```
┌─────────────┬────────────────────────────────────┐
│    参数     │               作用                 │
├─────────────┼────────────────────────────────────┤
│ temperature │ 控制随机性（0-2）                  │
│             │ 0 = 确定性高，1 = 平衡，2 = 创意高 │
├─────────────┼────────────────────────────────────┤
│ max_tokens  │ 限制输出长度                       │
├─────────────┼────────────────────────────────────┤
│ top_p       │ 控制采样范围（0-1）                │
├─────────────┼────────────────────────────────────┤
│ messages    │ 对话历史                           │
└─────────────┴────────────────────────────────────┘
```
