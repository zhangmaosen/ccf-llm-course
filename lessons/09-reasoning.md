## 🚀 课前思考

**AI 能解决复杂问题吗？**

```
问题：一个农夫有狼、羊、白菜，要过河
      船只能载农夫和一样东西
      狼会吃羊，羊会吃白菜
      怎么过河？

普通 AI：可能给出错误答案

推理 AI：（一步步分析）
第1步：带羊过河（狼不会吃白菜）
第2步：农夫独自返回
第3步：带狼过河
第4步：把羊带回来
第5步：带白菜过河
第6步：独自返回
第7步：带羊过河
完成！
```

**关键：** 推理能力让 AI 能解决更复杂的问题！

## 📖 第一部分：什么是推理？

### 🎬 视频导学
📺 [AI 推理能力详解](https://www.bilibili.com/video/BV1xxx)
🔗 [DeepSeek-R1 在线体验](https://chat.deepseek.com)

### 1.1 推理的定义

```
推理 = 从已知信息得出新结论的思维过程

人类推理：
────────────
看到乌云 → 推断要下雨 → 带伞出门
（经验 + 逻辑）

AI 推理：
────────────
输入问题 → 分析拆解 → 逐步推导 → 得出结论
```

### 1.2 推理的类型

```
┌─────────────────┬───────────────────────────────┐
│      类型       │            例子               │
├─────────────────┼───────────────────────────────┤
│  演绎推理       │ 所有人都会死 → 苏格拉底是人   │
│  （从一般到特殊）│ → 苏格拉底会死               │
├─────────────────┼───────────────────────────────┤
│  归纳推理       │ 看到很多天鹅是白的           │
│  （从特殊到一般）│ → 天鹅是白色的               │
├─────────────────┼───────────────────────────────┤
│  类比推理       │ A和B相似 → A有属性X          │
│  （从相似性出发）│ → B也可能有属性X             │
├─────────────────┼───────────────────────────────┤
│  逆向推理       │ 要得到结果 → 需要什么条件    │
│  （从结果倒推） │ → 从条件开始行动             │
└─────────────────┴───────────────────────────────┘
```

### 1.3 AI 推理 vs 人类推理

```
        人类推理                    AI 推理
           │                           │
    ┌──────┴──────┐             ┌──────┴──────┐
    │ 直觉 + 逻辑 │             │ 模式 + 计算 │
    │ 经验 + 创造 │             │ 训练 + 推导 │
    │ 情感 + 价值 │             │ 概率 + 优化 │
    └─────────────┘             └─────────────┘
    
人类优势：直觉、创造、情感
AI 优势：速度、规模、一致性
```

## 📖 第二部分：长思维链模型

### 2.1 短思维链 vs 长思维链

```
┌─────────────────────────────────────────────────┐
│              短思维链（普通 CoT）                 │
├─────────────────────────────────────────────────┤
│ 步骤少，快速给出答案                             │
│ 适合简单问题                                    │
│ 例：3-5 步推理                                  │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│              长思维链（Long CoT）                │
├─────────────────────────────────────────────────┤
│ 步骤多，深入分析问题                             │
│ 适合复杂问题                                    │
│ 例：几十甚至上百步推理                          │
│                                                 │
│ 代表模型：OpenAI o1、DeepSeek-R1               │
└─────────────────────────────────────────────────┘
```

### 2.2 测试时间扩展

```
训练时间优化 vs 测试时间优化

训练时间优化：
────────────────
花更多时间训练模型
模型更聪明，但训练成本高

测试时间扩展：
────────────────
让模型在回答时"多想一会"
不修改模型，给更多计算时间

类比：
训练时间 = 上课时认真听讲
测试时间 = 考试时仔细检查
```

### 2.3 推理模型的能力

```
┌─────────────────┬───────────────────────────────┐
│     任务类型    │            例子               │
├─────────────────┼───────────────────────────────┤
│  数学问题       │ 奥数题、复杂计算              │
├─────────────────┼───────────────────────────────┤
│  逻辑推理       │ 谜题、逻辑难题                │
├─────────────────┼───────────────────────────────┤
│  代码编写       │ 复杂算法、调试                │
├─────────────────┼───────────────────────────────┤
│  科学问题       │ 物理推理、化学方程            │
├─────────────────┼───────────────────────────────┤
│  规划问题       │ 旅行规划、项目计划            │
└─────────────────┴───────────────────────────────┘
```

## 📖 第三部分：推理模型的训练

### 3.1 基于监督微调

```
训练流程：

第1步：收集带推理步骤的数据
─────────────────────────────
问题 + 详细的思考过程 + 答案

第2步：让模型学习这些推理模式
─────────────────────────────
学会"怎么一步步思考"

第3步：在测试时应用
─────────────────────────────
面对新问题，也能给出推理过程
```

### 3.2 基于强化学习

```
RL 训练推理模型：

┌─────────────────────────────────────────────────┐
│                                                 │
│  问题 ──→ 模型生成推理 ──→ 检查答案            │
│                ↑              │                 │
│                │              ↓                 │
│                └──── 奖励信号 ←┘                │
│                                                 │
│  答案对 → 正向奖励 → 鼓励这种推理              │
│  答案错 → 负向奖励 → 避免这种推理              │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 3.3 过程奖励 vs 结果奖励

```
结果奖励（Outcome Reward）：
────────────────────────────
只看最终答案对不对
简单，但可能学到错误的推理

过程奖励（Process Reward）：
────────────────────────────
检查每一步是否正确
更精确，但需要更多标注

类比：
结果奖励 = 只看考试分数
过程奖励 = 看每道题的解题过程
```

## 📖 第四部分：搜索增强推理

### 4.1 自一致性（Self-Consistency）

```
方法：多次生成答案，选最常见的

同一个问题，让 AI 回答 10 次：
答案1: A
答案2: A
答案3: B
答案4: A
答案5: A
...

多数选 A → 最终答案选 A

效果：提高准确率
```

### 4.2 思维树（Tree of Thoughts）

```
把推理过程组织成树结构：

                    问题
                     │
          ┌──────────┼──────────┐
          ↓          ↓          ↓
        思路A      思路B      思路C
          │          │          │
       ┌──┴──┐    ┌──┴──┐    ┌──┴──┐
       ↓     ↓    ↓     ↓    ↓     ↓
      A1   A2    B1   B2    C1   C2

探索多条路径，找到最优解
```

### 4.3 搜索方法对比

```
┌─────────────┬────────────────────────────────┐
│    方法     │              说明              │
├─────────────┼────────────────────────────────┤
│ 贪心搜索   │ 只走一条路，可能错过最优解     │
├─────────────┼────────────────────────────────┤
│ 自一致性   │ 多次尝试，投票选择             │
├─────────────┼────────────────────────────────┤
│ 思维树     │ 探索多条路径，系统搜索         │
├─────────────┼────────────────────────────────┤
│ 蒙特卡洛   │ 随机采样，高效探索             │
│ 树搜索     │ （用于围棋 AI）                │
└─────────────┴────────────────────────────────┘
```

## 🔬 课堂实验

### 实验 1：体验推理模型（10分钟）

**用普通模型和推理模型解决同一问题：**

问题：`"一个袋子里有红球和蓝球，红球是蓝球的2倍。取出一个红球后，红球和蓝球数量相等。问原来各有多少个球？"`

对比：
- 普通模型的回答
- 推理模型（如 DeepSeek-R1）的回答

### 实验 2：思维链实践（5分钟）

**尝试用思维链解决这道题：**

```
小明的年龄是小红的2倍，5年后小明比小红大15岁。
问：现在小明和小红各多少岁？
```
