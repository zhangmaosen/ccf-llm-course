## 📖 第一部分：为什么需要评测？

### 1.1 评测的目的

```
┌─────────────────────────────────────────────────┐
│              模型评测的目的                      │
├─────────────────────────────────────────────────┤
│                                                 │
│  📈 比较不同模型                                │
│     谁更强？谁更适合某个任务？                  │
│                                                 │
│  🔍 发现模型问题                                │
│     模型有什么不足？需要改进什么？              │
│                                                 │
│  📏 评估改进效果                                │
│     新版本比旧版本好多少？                      │
│                                                 │
│  🎯 选择合适的模型                              │
│     对于我的任务，该用哪个模型？                │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 1.2 评测的挑战

```
大模型评测的困难：

❌ 任务多样
   翻译、写作、编程、推理...用不同的指标

❌ 主观性强
   "写得好不好"很难量化

❌ 长尾问题
   常见问题表现好，罕见问题可能不行

❌ 数据污染
   测试数据可能被用于训练

❌ 泛化能力
   在测试集好 ≠ 在真实场景好
```

## 📖 第二部分：评测指标

### 2.1 分类任务指标

**混淆矩阵：**

```
                预测结果
              正类    负类
         ┌────────┬────────┐
实  正类 │   TP   │   FN   │
际      ├────────┼────────┤
    负类 │   FP   │   TN   │
         └────────┴────────┘

TP (True Positive)：正确预测为正类
FP (False Positive)：错误预测为正类
FN (False Negative)：错误预测为负类
TN (True Negative)：正确预测为负类
```

```
                预测结果
              正类    负类
         ┌────────┬────────┐
实  正类 │   TP   │   FN   │
际      ├────────┼────────┤
    负类 │   FP   │   TN   │
         └────────┴────────┘

TP (True Positive)：正确预测为正类
FP (False Positive)：错误预测为正类
FN (False Negative)：错误预测为负类
TN (True Negative)：正确预测为负类
```

**核心指标：**

```
┌─────────────┬─────────────────────┬─────────────────┐
│    指标     │        公式         │      含义       │
├─────────────┼─────────────────────┼─────────────────┤
│  准确率     │ (TP+TN) / 总数      │ 整体对不对      │
│  Accuracy   │                     │                 │
├─────────────┼─────────────────────┼─────────────────┤
│  精确率     │ TP / (TP + FP)      │ 预测正的有多少  │
│  Precision  │                     │ 是真的正        │
├─────────────┼─────────────────────┼─────────────────┤
│  召回率     │ TP / (TP + FN)      │ 真正的有多少    │
│  Recall     │                     │ 被找出来        │
├─────────────┼─────────────────────┼─────────────────┤
│  F1 分数    │ 2×P×R / (P+R)       │ 精确率和召回率  │
│             │                     │ 的调和平均      │
└─────────────┴─────────────────────┴─────────────────┘
```

**例子：垃圾邮件检测**

```
有 100 封邮件，20 封是垃圾邮件

检测结果：
- 正确识别垃圾邮件：18 封
- 误判为垃圾邮件：5 封
- 漏掉垃圾邮件：2 封

计算：
- TP = 18, FP = 5, FN = 2, TN = 75
- 精确率 = 18 / (18+5) = 78%
- 召回率 = 18 / (18+2) = 90%
- 准确率 = (18+75) / 100 = 93%
```

```
┌─────────────┬─────────────────────────────────────┐
│    指标     │                 说明                │
├─────────────┼─────────────────────────────────────┤
│  BLEU       │ 机器翻译质量指标                    │

│             │ 用向量计算文本相似程度              │
└─────────────┴─────────────────────────────────────┘
```

### 2.3 排序任务指标

```
NDCG（归一化折损累积增益）

用于评估搜索结果、推荐系统的排序质量

考虑因素：
- 相关性高的排在前面，得分更高
- 排名越靠后，得分折扣越大
- 最终归一化到 0-1 范围
```

## 📖 第四部分：评测方法

### 4.1 自动评测

```
方法：用程序自动计算指标

优点：
✅ 快速
✅ 可重复
✅ 客观

缺点：
❌ 可能与人类感受不一致
❌ 无法评估创意性
```

### 4.2 人类评测

```
方法：让人评价模型输出

评价维度：
- 相关性：回答是否相关
- 准确性：信息是否正确
- 流畅性：语言是否通顺
- 有用性：是否有帮助

优点：
✅ 更贴近真实体验
✅ 能评估复杂任务

缺点：
❌ 昂贵
❌ 主观
❌ 慢
```

### 4.3 模型评测模型

```
方法：用强大的模型（如 GPT-4）评价其他模型

流程：
────────────────
1. 让被测模型生成回答
2. 让评测模型打分
3. 汇总分数

优点：
✅ 比人类便宜
✅ 比自动评测更智能

缺点：
❌ 评测模型可能有偏见
❌ 成本仍然较高
```
