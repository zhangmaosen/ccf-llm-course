## 📖 第一部分：AI 偏见

### 1.1 什么是偏见？

```
AI 偏见：AI 系统对某些群体不公平的对待

表现：
────────────────
- 刻板印象：认为护士是女性，工程师是男性
- 歧视输出：对某些群体使用负面描述
- 不公平决策：在贷款、招聘中歧视某些群体
```

### 1.2 偏见的来源

```
┌─────────────────────────────────────────────────┐
│              偏见的来源                          │
├─────────────────────────────────────────────────┤
│                                                 │
│  📊 数据偏见                                    │
│     训练数据本身有偏见                          │
│     例：新闻报道中某些群体出现频率低            │
│                                                 │
│  🔧 算法偏见                                    │
│     模型设计有缺陷                              │
│     例：过度依赖某些特征                        │
│                                                 │
│  🌍 社会偏见                                    │
│     反映了现实社会的偏见                        │
│     例：历史数据中存在的歧视                    │
│                                                 │
│  📝 标注偏见                                    │
│     人工标注时的偏见                            │
│     例：标注者的价值观影响标签                  │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 1.3 偏见的例子

```
┌─────────────────┬───────────────────────────────┐
│     领域        │            偏见例子           │
├─────────────────┼───────────────────────────────┤
│  性别偏见       │ "医生"默认用"他"             │
│                 │ "护士"默认用"她"             │
├─────────────────┼───────────────────────────────┤
│  种族偏见       │ 对某些族群的刻板描述         │
│                 │ 在犯罪预测中歧视某些群体     │
├─────────────────┼───────────────────────────────┤
│  文化偏见       │ 西方视角为主                 │
│                 │ "龙"在东西方含义不同         │
├─────────────────┼───────────────────────────────┤
│  年龄偏见       │ 对老年人的刻板印象           │
│                 │ "老人 = 不懂技术"            │
└─────────────────┴───────────────────────────────┘
```

### 1.4 偏见的危害

```
对个人的危害：
────────────────
- 不公平对待
- 机会丧失
- 尊严受损

对社会的危害：
────────────────
- 加剧不平等
- 固化刻板印象
- 社会信任下降
```

### 1.5 如何缓解偏见？

```
方法 1：数据平衡
────────────────
确保训练数据多样化、平衡

方法 2：去偏见训练
────────────────
在训练中减少偏见学习

方法 3：人工审核
────────────────
人工检查和纠正偏见输出

方法 4：用户教育
────────────────
让用户了解 AI 可能有偏见

方法 5：持续监控
────────────────
定期检测偏见问题
```

## 📖 第二部分：AI 安全

### 2.1 安全风险类型

```
┌─────────────────────────────────────────────────┐
│              AI 安全风险                         │
├─────────────────────────────────────────────────┤
│                                                 │
│  🔒 隐私泄露                                    │
│     AI 可能泄露训练数据中的隐私信息             │
│                                                 │
│  📰 虚假信息                                    │
│     AI 生成不真实的内容                         │
│                                                 │
│  🎭 深度伪造                                    │
│     AI 生成逼真的假视频、音频                   │
│                                                 │
│  ⚠️ 恶意使用                                   │
│     被用于诈骗、攻击等                          │
│                                                 │
│  🎯 社会工程                                    │
│     被用于钓鱼攻击、社会操控                    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 2.2 隐私保护

```
AI 隐私问题：
────────────────
- 训练数据可能包含敏感信息
- 对话中可能泄露隐私
- 模型可能"记住"隐私数据

保护方法：
────────────────
┌─────────────────┬───────────────────────────────┐
│      方法       │              说明              │
├─────────────────┼───────────────────────────────┤
│  数据脱敏       │ 移除或加密敏感信息            │
├─────────────────┼───────────────────────────────┤
│  差分隐私       │ 添加噪声保护个体隐私          │
├─────────────────┼───────────────────────────────┤
│  联邦学习       │ 数据不离开本地                │
├─────────────────┼───────────────────────────────┤
│  用户控制       │ 让用户控制自己的数据          │
└─────────────────┴───────────────────────────────┘
```

### 2.3 虚假信息

```
问题：AI 生成的虚假信息

类型：
────────────────
- 幻象（Hallucination）：AI 编造不存在的信息
- 深度伪造：AI 生成假的图片、视频
- 虚假新闻：AI 批量生成假新闻

危害：
────────────────
- 误导公众
- 损害名誉
- 影响选举
- 扰乱社会

应对：
────────────────
- 提高公众识别能力
- 技术检测 AI 生成内容
- 建立溯源机制
```

### 2.4 恶意使用

```
AI 可能被恶意使用的方式：

🚫 生成恶意代码
   编写病毒、木马程序

🚫 社会工程攻击
   生成逼真的钓鱼邮件

🚫 批量生成虚假内容
   操控舆论、刷评

🚫 自动化攻击
   批量破解密码、攻击系统
```

## 📖 第三部分：负责任地使用 AI

### 3.1 使用者的责任

```
作为 AI 用户，你应该：

✅ 验证信息
   不要盲目相信 AI 的回答
   重要信息要交叉验证

✅ 保护隐私
   不要输入敏感个人信息
   注意隐私设置

✅ 遵守法律
   不用于非法目的
   尊重知识产权

✅ 诚实使用
   标明使用了 AI
   不把 AI 内容当原创

✅ 举报问题
   发现有害内容要举报
   帮助改进 AI
```

### 3.2 开发者的责任

```
作为 AI 开发者，应该：

✅ 安全设计
   在设计阶段考虑安全问题

✅ 测试验证
   充分测试，发现潜在问题

✅ 透明说明
   告知用户 AI 的局限性

✅ 持续改进
   根据反馈不断优化

✅ 应急响应
   建立问题发现和处理机制
```

### 3.3 AI 使用的原则

```
┌─────────────────────────────────────────────────┐
│            负责任 AI 使用原则                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  🔍 透明性                                      │
│     清楚说明使用了 AI                           │
│                                                 │
│  ⚖️ 公平性                                      │
│     不歧视任何群体                              │
│                                                 │
│  🔒 隐私性                                      │
│     保护用户隐私                                │
│                                                 │
│  🛡️ 安全性                                      │
│     确保系统安全                                │
│                                                 │
│  📊 可问责性                                    │
│     对 AI 行为负责                              │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 🔬 课堂实验

### 实验 1：识别偏见（10分钟）

**分析以下 AI 输出是否存在偏见：**

```
输出 1："工程师正在和他的团队开会"
问题：是否有性别偏见？

输出 2："这个地区的犯罪率很高"
问题：是否可能有地区偏见？

输出 3："像你这样的年轻人应该很懂电脑"
问题：是否存在刻板印象？
```

- 使用 AI 时应该注意什么
- 哪些行为是不允许的
